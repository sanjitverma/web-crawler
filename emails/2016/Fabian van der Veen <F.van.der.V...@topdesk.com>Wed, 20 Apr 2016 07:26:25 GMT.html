Hey,

The way you describe it; this sounds like a problem I also encountered in our (similar sized)
project here.
I reported this here: https://issues.apache.org/jira/browse/MNG-5960, which has been linked
as related to https://issues.apache.org/jira/browse/MNG-5750.

Troubleshooting boiled down to running maven in debug mode; which might be something you don't
want to do.
In any case, I found that plugins running aggregate goals (e.g. javadoc:aggregate) will resolve
(compile) dependencies for _all_ projects in the reactor. Running such a plugin in anything
else than your root pom might cause the random failures you have. (And having the plugin in
your root pom might also effectively include its configuration and execution in child poms;
which is "debuggable" using help:effective-pom.)

Hopefully this helps you.

- Fabian
