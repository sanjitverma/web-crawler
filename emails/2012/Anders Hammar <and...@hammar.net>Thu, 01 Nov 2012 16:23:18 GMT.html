> > The line build.xml(250) is where ...MavenCLI class gets invoked/forked

>
> Line 250 in build.xml is where it forks the JVM - if you capture the
> output from running "ant -d -f build.xml" and look for "maven-compile:"
> then it should show the actual command it forks (ie. the one that returns
> error code 1). You could copy that command and all its arguments (ie. java
> -classpath ...<snip>... org.apache.maven.cli.MavenCli ...<etc>...) and try
> it from the command-line to see if there's an error message that's not
> getting passed on. Perhaps also add '-X' after
> 'org.apache.maven.cli.MavenCli' to enable full debug output.
>
> > OK - we sit behind a firewall , if that makes a difference?
>
> The bootstrap build should use your local settings.xml just like Maven, so
> as long this is configured with any proxy needed to access Maven Central
> then it should be fine.
