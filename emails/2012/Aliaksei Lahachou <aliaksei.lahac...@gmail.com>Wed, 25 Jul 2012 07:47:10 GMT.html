Hi Mirko!

You may try doing the same trick as the cobertura-maven-plugin does when
aggregate is set to true.

1. Every module executes the plugin, but the plugin updates the same file
in the directory of the project from which the build was started. In you
case you may, for example, calculate SHA1 of the module artifacts and
append them to a file.
2. When the execution belongs to the last module in the reactor,
cobertura-maven-plugin generates a report in the directory of the project
from which the build was started. At this point you have all artifacts, the
list of SHA1, and you may do other stuff with it, for example, post the
list somewhere.
