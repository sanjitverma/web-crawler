Simone Tripodi wrote:

> Salut Jeff,
> 
> profiling sounds a good idea but I didn't understand how mvn is able
> to resolve profiled dependencies, I mean, in my project X I need to
> import the c.jar dependency with mac-x86_64 as classifier - how to
> guarantee transitive dependencies are imported for the same platform
> (same classifier)?

Maybe you take still a loot at the NAR plugin. It is not only able to build 
shared libraries (or DLLs), it manages them also as dependencies using a 
'NAR archive'. It's no requirement that you have built your shared libs with 
it, but you may create the NAR archive manually (well, using the assembly 
plugin) to use its dependency management. Have a look at its JNI example and 
how a later project refers the artifacts. The plugin uses the classifier to 
"encode" the combination of  OS/CPU/Compiler.
